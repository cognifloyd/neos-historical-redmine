Task #38755: Usability issues collected on the UX sprint

Swap doc header rows in all backed modules

Added by Steffen Ritter about 3 years ago. Updated about 3 years ago.

Status:Resolved Start date:2012-07-08
Priority:Should have Due date:
Assigned To:- % Done:

100%
Category:Backend User Interface Spent time: -
Target version:6.0.0-pre-beta1
TYPO3 Version:6.0 Complexity:easy
PHP Version: Sprint Focus:

Description

The contents of the do headers need to be adjusted.
Module relevant things ( like the submodule switch) should be on top while the content related things (save icon for example) should be in the second row - near the content.

Missing do headers should be added to the modules lacking them...

Revision 703bca18
Added by Felix Kopp about 3 years ago

[FEATURE] Swap doc header rows in all backend modules

Swaps all docHeader rows for the backend modules.

Function menu and CSH are moved to the top row while
buttons (save, close and shortcut) are moved underneath.

Also includes a compatibility layer for non-core legacy
extensions that don't include expected template structure.
These non-standard docHeaders are reformated by CSS.
To deliver fall-back logic the HTML structure was redefined.
