Feature #49949

Respect restricted roles for modules

Added by Aske Ertmann about 2 years ago. Updated about 2 years ago.

Status:Resolved Start date:2013-07-11
Priority:Should have Due date:
Assigned To:Aske Ertmann % Done:

100%
Category:-
Target version:Base Distribution - 1.0 alpha 5

Description

  • It should be possible to configure a role for submodules
  • The menu component should respect the role restriction
  • The module controller should prevent accessing a restricted role without having the necessary role

Associated revisions

Revision 400160b6
Added by Aske Ertmann about 2 years ago

[!!!][FEATURE] Restrict access to specific modules

Additionally adds a version parameter to the menu
controller request, to ensure that changes in roles
will update the accessible modules accordingly.

This is breaking since it's no longer possible to use
the standard module controller if no controller is
specified. This is done to be able to enforce restrictions
on overview modules with resource policies.
