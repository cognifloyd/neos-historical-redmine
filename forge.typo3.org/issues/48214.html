Bug #48214

Moving of nodes across levels should not show the same node twice

Added by Sebastian Kurfuerst about 2 years ago. Updated almost 2 years ago.

Status:Resolved Start date:2013-05-15
Priority:Should have Due date:
Assigned To:Sebastian Kurfuerst % Done:

100%
Category:-
Target version:Base Distribution - 1.0 beta 1

Description

If having a node structure in live workspace, and MOVING a node into another
parent node, this is copied into your personal workspace.

In neos, the moved node is then shown twice: First from the user workspace (correct), and from the live workspace (wrong!).

It only breaks for fetching collections of nodes, but that's what we do in Neos all the time.
