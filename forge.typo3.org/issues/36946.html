Bug #46389: Faulty Workspace Placeholders

Copy paste in live makes draft placeholders visible.

Description

When doing a copy paste of a page and subelements in live, all placeholders (initial, move-to) will show up in the copy.
Frontend rendering will throw an error in case of tt_content placeholders.

Holds for 4.5, 4.6 and 4.7 (didn't check 6.0)

Steps to reproduce:

1. Create a page in the live workspace.
2. Open a draft workspace and add a subpage to the page created in 1.
3. Move back to Live and copy the page created in 1. with recursive copying turned on.
4. The placeholder for the draft subpage will now become visible under the copy of the page.

Same goes for tt_content, in this case frontend output will also be broken.
