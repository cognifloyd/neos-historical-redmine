Task #38755: Usability issues collected on the UX sprint

Remove "switch to list" in doc-header of the page module

Added by Steffen Ritter about 3 years ago. Updated about 3 years ago.

Status:Resolved Start date:2012-07-10
Priority:Should have Due date:
Assigned To:- % Done:

100%
Category:Backend User Interface Spent time: -
Target version:6.0.0-alpha3
TYPO3 Version:6.0 Is Regression:
PHP Version: Sprint Focus:
Complexity:

Description

In the upper right corner of the docheader at the page module there is a small button to switch to the list module.
This is redundant and as well takes not care of the moduleMenu and other backend states which leads to an inconsistent backend state.

just remove the icon.

Associated revisions

Revision 40871b96
Added by Felix Kopp about 3 years ago

[TASK] Remove record_list icon from Page module

Removes the record_list icon from the docHeader in Page module.

This icon links from page to list module but does not bring the
backend to a consistent state. The module bar is e.g. not updated.

The decision to remove the icon was taken at UX sprint 2012.
